# 个人简历

简历是申请材料中最高频被阅读的一份。它应该服务于一个目标：让老师在很短时间内抓到你的优势与方向匹配度。

## 推荐结构

- 基本信息：姓名、学校、专业、联系方式、邮箱。
- 教育背景：绩点、排名、核心课程、英语成绩。
- 科研经历：课题背景、你的工作、方法、结果和产出。
- 项目经历：问题、方案、技术栈、个人贡献和效果。
- 竞赛与荣誉：优先放与申请方向相关或含金量高的项目。
- 技能与其他：编程、实验技能、工具、语言能力等。

## 写作原则

- 用动词开头描述贡献，例如“构建”“实现”“验证”“分析”。
- 尽量量化结果，例如排名、数据规模、性能提升、奖项等级。
- 不夸大个人贡献，面试时讲不清楚的内容不要写得过满。
- 针对不同方向微调顺序，把最相关经历放在前面。

## 常见问题

- 经历少怎么办：突出课程项目、课程论文、实验报告和自主学习成果。
- 奖项少怎么办：把科研、项目、排名、英语等其他优势表达清楚。
- 是否放照片：按院校要求来；没有要求时保持简洁专业即可。
